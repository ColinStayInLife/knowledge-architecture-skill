# 信息管理架构详解

## 核心理念

### 1. 信息流动

```
输入 → 处理 → 输出 → 沉淀 → 调用
```

- **输入**：碎片信息通过 AI 助手收集
- **处理**：AI 自动整理、分类、摘要
- **输出**：日报、项目文档、知识笔记
- **沉淀**：长期记忆、知识库
- **调用**：AI 可随时读取历史知识

### 2. 双层记忆

#### 短期记忆（Daily Log）

- 位置：`memory/YYYY-MM-DD.md`
- 自动记录每日对话
- 定期清理

#### 长期记忆（MEMORY.md）

- 位置：工作区根目录
- 重要信息永久保存
- AI 每次对话都会读取

### 3. 知识库结构

采用 PARA 方法 + Johnny Decimal：

| 目录 | 编号 | 说明 |
|------|------|------|
| Inbox | 00 | 快速收集 |
| Daily | 01 | 日报/日志 |
| Projects | 02 | 活跃项目 |
| Knowledge | 03 | 长期知识 |
| Resources | 04 | 参考资料 |
| Archive | 05 | 归档 |
| System | 99 | 系统/模板 |

### 4. AI 联动

通过软链接实现：

- 在 Obsidian 中编辑 AI 配置
- AI 的记忆与知识库打通
- 对话自动整理进 OB

## 最佳实践

1. **每天说"整理今天的工作"** - 触发自动整理
2. **重要信息手动更新 MEMORY.md** - 长期保存
3. **项目进展及时更新** - 保持项目概览最新
4. **定期回顾** - 每周回顾日报，更新长期记忆
